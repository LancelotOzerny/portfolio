# Задачи

## `[M000]` показ представлений
**Задача:**
Реализовать показ представлений с помощью текущего класса контроллера. 
Показ представлений должен включать в себя следующий функционал:
- Выбор представления из соответствующей директории контроллера
- Вывод представления с передачей аргументов

## `[M001]` Вывод скриптов и стилей
**Задача:**
Реализовать подключение к странице дополнительных скриптов и стилей. Требуется написать:
- подключение скриптов и стилей из публичных директорий
- Подключение скриптов и стилей из директории на уровень выше публичной (компоненты, шаблоны)
- Скрипты и стили должны подключаться к странице автоматически в bootstrap.php

## `[M002]` Реализация шаблонной системы
**Задача:**
Реализовать шаблонную систему сайта, которая позволит подключать множество шаблонов 
по заранее определенным путям в файлах конфигурации. Функционал должен содержать:
- Выбор шаблона (Default если никакой не выбран)
- Подключение шапки и подвала через методы
- Задача каких-либо параметров (title, meta и др)

## `C000` Навигационное меню
**Задача:**
Реализовать компонент навигационного меню, который позволит управлять текущим меню через файлы конфигурации
(к примеру ```/App/Config/Navigaiton/main.php```) и выводить одноуровневое меню на сайте в определенном шаблоном месте

## `[M003]` Авторизация
**Задача:**
Реализовать авторизацию для административной части, которая позволит войти в систему под правильными
логином и паролем. Форма авторизации должна содержать:
- Заголовок
- Логин
- Пароль
- Кнопку авторизоваться

## `[C001]` Вывод проектов на экран
{`[M003]`}

**Задача:**
Получение списка проектов и вывод их на экран. Список выводится в виде карточек. Проект содержит:
- id \[unsigned int]
- created_at \[timestamp]
- changed_at \[timestamp]
- preview_text \[varchar (255)]
- preview_img_url \[varchar (255)]
- detail_text \[varchar (255)]
- detail_img_url \[varchar (255)]

К проекту прикрепляются теги и ссылки.
Теги и проекты соединены связью многие ко многим и содержат поля:
- id \[unsigned int]
- created_at \[timestamp]
- changed_at \[timestamp]
- name \[varchar (255)]

Ссылки и проекты соединены связью один ко многим и содержат поля:
- id \[unsigned int]
- created_at \[timestamp]
- changed_at \[timestamp]
- name \[varchar (255)]
- url \[varchar (255)]
- project_id \[unsigned int]

На выходе должен получится модуль, который позволяет:
* Управлять проектами (CRUD) в административной части
  * Вывод списка проектов в виде плиток
  * Создание нового проекта
  * Редактирование определенного проекта
  * Удаление определенного проекта с подтверждением
* Выводить компонент, который обрабатывает проекты и выдает их в виде списка объектов
* Автоматически установить требуемые таблицы если их нет
